# OneDayTrading 인스타 뉴스 카드 자동화 로직

## 1. 전체 파이프라인

```text
신규 기사 감지
→ 기사 원문 HTML 직접 조회
→ 메타데이터/본문/대표 이미지 추출
→ 기사 핵심 대상과 국가·시장 분류
→ 대표 이미지 사용 가능성 판단
→ 기사 전용 무문자 이미지 생성 또는 원본 이미지 재구성
→ 뉴스 카피 생성 및 사실 검증
→ 1080×1350 카드 4장 렌더링
→ 로고·출처·CTA 후처리
→ 검수
→ 게시 승인/예약
```

이미지 생성 모델에는 글자를 절대 맡기지 않는다. 이미지 모델은 사진 또는 묘사만 만들고, 모든 한글·숫자·로고·출처는 렌더러가 마지막에 삽입한다.

## 2. 기사 수집

신규 기사 URL을 발견하면 검색 결과나 유사 기사로 내용을 추측하지 않고 해당 URL의 HTML을 직접 조회한다.

필수 추출값:

- `article_url`
- `title`
- `description`
- `body`
- `published_at`
- `source_name`
- `og_image`
- `category`
- `sector`
- `entities[]`: 인물, 기업, 종목, 제품, 장소, 작품
- `sentiment`: 호재, 악재, 중립

`article_url` 또는 기사 ID가 이미 처리된 경우 중복 생성하지 않는다. 제목만 같은 기사는 URL과 게시 시각까지 비교한다.

## 3. 시장 라벨 결정

```text
한국 거래소 종목·한국 증시 중심 → 국내주식
미국 및 기타 해외 거래소 종목 중심 → 해외주식
국내외 종목이 동시에 핵심 → 글로벌증시
종목 연결이 약한 거시경제 기사 → 증시·경제
가상자산 중심 → 크립토
```

하단 고정 표기:

```text
onedaytrading.net | {market_label}
```

예시:

```text
onedaytrading.net | 해외주식
onedaytrading.net | 국내주식
```

## 4. 이미지 선택 및 생성 로직

### 4.1 핵심 대상 우선순위

1. 기사에 명시된 핵심 인물
2. 핵심 제품·게임·자동차·시설·장소
3. 기업의 사업 현장 또는 사건을 상징하는 구체적인 장면
4. 위 대상이 없을 때만 업종을 설명하는 에디토리얼 장면

범용 차트, 지구본, 돈다발, 악수, 의미 없는 트레이더 화면은 사용하지 않는다.

### 4.2 대표 이미지 처리

```text
대표 이미지가 기사 핵심과 정확히 일치
  → 구도·인물·색감을 참고해 세로형 무문자 이미지로 재구성

대표 이미지의 사용 권한과 재가공 권한이 확인됨
  → 원본을 4:5로 확장/크롭하고 텍스트 영역을 확보

대표 이미지가 부정확하거나 품질이 낮음
  → 본문의 핵심 대상만 사용해 새로운 에디토리얼 이미지 생성

실존 인물이 기사 핵심
  → 기사 대표 이미지 또는 신뢰 가능한 인물 사진을 정체성 참고 이미지로 사용

기사 핵심 대상이 불명확
  → 자동 게시 금지, 수동 검수 큐로 이동
```

### 4.3 이미지 생성 프롬프트 필수 조건

- 세로형 `4:5`, 최종 렌더 크기 `1080×1350`
- 기사 핵심 대상과 사건을 프롬프트 첫 부분에 명시
- 인물 또는 핵심 대상은 오른쪽이나 중앙 하단 배치
- 제목이 올라갈 왼쪽 상단은 어둡고 단순한 여백으로 확보
- 하단에는 로고와 사이트 주소를 넣을 안전 영역 확보
- 에디토리얼 사진, 자연스러운 피부와 실제 공간 질감
- `no text, no letters, no numbers, no logo, no watermark`
- 과장된 사이버펑크, 플라스틱 피부, 범용 금융 그래픽 금지

## 5. 카피 생성 규칙

### 뉴스 타이틀

- 권장 18~32자, 최대 2~3줄
- 핵심 인물·기업·사건을 반드시 포함
- 원문 제목을 그대로 복사하기보다 의미를 보존해 압축
- 확인되지 않은 단정, 선정적 표현, 매수·매도 권유 금지

### 서브타이틀

- 권장 28~55자, 최대 2줄
- “왜 중요한가” 또는 “기업·주가에 어떤 변수인가”를 설명
- 타이틀과 같은 말을 반복하지 않는다

### CTA

기본값:

```text
핵심 분석 보기 →
```

기사 유형별 분기:

```text
분석·전망 중심     → 핵심 분석 보기 →
단순 속보·사실 중심 → 기사 자세히 보기 →
인물·인터뷰 중심   → 전체 내용 보기 →
데이터·공시 중심   → 투자 포인트 보기 →
```

`상세보기 >>>`보다 `핵심 분석 보기 →`를 권장한다. 의미가 구체적이고 뉴스 브랜드의 분석 성격을 전달하며, 화살표 하나가 더 정돈돼 보인다.

## 6. 1페이지 레이아웃

```text
┌──────────────────────────┐
│ [속보/호재/악재/중립]     │
│                          │
│ 뉴스 타이틀              │
│ 최대 2~3줄               │
│                          │
│ 서브타이틀 최대 2줄       │
│                          │
│           핵심 인물/장면  │
│                          │
│             핵심 분석 보기 →│
│ NEWLOGO                  │
│ onedaytrading.net | 해외주식│
└──────────────────────────┘
```

- 바깥 안전 여백: 72px
- 상단 상태 라벨: 좌측 상단
- 제목: 좌측 정렬, 가장 높은 시각 우선순위
- 서브타이틀: 제목 아래 24~32px 간격
- CTA: 오른쪽 하단, 사이트 주소보다 위쪽에 작게 표시
- `NEWLOGO`: 왼쪽 하단
- 사이트·시장 라벨: 하단 기준선에 배치
- 인물 얼굴, 눈, 핵심 제품을 글자로 가리지 않는다
- 배경이 밝으면 하단과 제목 영역에만 국소 그라데이션을 적용한다

## 7. 2~4페이지 구성

### 2페이지 — 핵심 정리

- 1페이지와 동일한 배경 이미지와 색상 토큰을 사용하되, 80% 이상 어둡게 덮어 읽기 화면으로 전환
- 인포그래픽 카드 박스는 사용하지 않고 실제 경제지 분석면처럼 연속된 기사 레이아웃 사용
- 상단에 `MARKET ANALYSIS · 02`, 기사형 헤드라인, 파란 구분선, 리드문, 편집팀 바이라인 배치
- 전망, 추가 확인 정보, 확장 가능성을 별도 제목이나 카드로 나누지 않고 하나의 분석 기사로 통합
- 본문 흐름은 `기대 효과 → 확인할 변수 → 확장 가능성` 순서의 3문단으로 구성
- 본문 왼쪽에는 얇은 파란 세로선을 사용하고 마지막에 핵심 결론 한 줄 배치
- 하단에는 `NEWLOGO`, `onedaytrading.net | {market_label}`, `02 / 04` 고정

### 3페이지 — 분석

- 논의: 시장에서 엇갈리는 쟁점
- 접근: 투자자가 확인할 데이터·일정·조건
- 전망: 단기와 중기 시나리오

각 항목은 2~3문장 이내로 제한한다.

### 4페이지 — 브랜드 CTA

- `더 많은 증권·증시 소식은?`
- 인스타그램 계정명
- `매일 업데이트되는 핵심 뉴스를 확인하세요.`
- `onedaytrading.net`
- `NEWLOGO`
- 투자 참고용이며 매매 권유가 아니라는 고지

## 9. 자동 검수 조건

아래 중 하나라도 실패하면 자동 게시하지 않는다.

- 생성 이미지의 핵심 대상이 기사와 일치하는가
- 인물 이름과 얼굴 참고 이미지가 일치하는가
- 제목의 기업·종목·숫자가 본문에서 확인되는가
- 국내주식/해외주식 라벨이 거래 시장과 일치하는가
- 이미지에 AI가 만든 글자·로고·워터마크가 없는가
- 텍스트가 얼굴과 핵심 피사체를 가리지 않는가
- 제목과 서브타이틀이 잘리지 않는가
- 원문 출처와 기사 URL이 저장됐는가
- 투자 권유로 해석될 표현이 없는가
