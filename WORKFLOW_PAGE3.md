# OneDayTrading 세 번째 페이지 워크플로우

## 목적

기사의 투자자 관점 종합 평가와 그 판단의 핵심 근거를 전달한다. 단순 요약을 반복하지 않고, “현재 무엇을 긍정·부정적으로 볼 수 있는지”와 “최종 판단을 바꿀 조건은 무엇인지”를 명확히 설명한다.

이 페이지는 오직 `그래서 어떻게 평가해야 하나?`에 답한다. 2페이지의 배경 설명을 반복하지 않고 `기대할 것 / 확인할 것 / 판단할 것` 세 기준으로 압축한다. 세부 생성 규칙은 `ENGAGEMENT_ALGORITHM.md`를 따른다.

## 입력 데이터


## 평가 분류

```text
명확한 호재   → 실적·수주·매출 효과가 수치로 확인됨
제한적 호재   → 방향은 긍정적이나 재무 효과가 아직 확인되지 않음
중립          → 긍정·부정 요인이 함께 존재하거나 영향이 작음
제한적 악재   → 단기 부담은 있으나 회복 조건이 존재함
명확한 악재   → 손실·규제·계약 취소 등 부정 영향이 확인됨
```

평가는 기사 제목의 감성 라벨을 그대로 복사하지 않고, 기사 본문과 공개된 근거를 바탕으로 다시 결정한다.

## 콘텐츠 생성 구조

1. 평가형 헤드라인: 핵심 판단을 질문 또는 결론 형태로 최대 2줄
2. 평가 라벨: `제한적 호재`처럼 강도를 포함한 짧은 표현
3. 종합 평가: 현재 시점의 판단을 2문장으로 설명
4. 핵심 내용: 다음 세 요소를 연속된 기사 본문으로 작성
   - 긍정 요인
   - 위험·불확실성
   - 판단을 강화하거나 뒤집을 확인 조건
5. 최종 평가: 한 줄 결론

`긍정 요인 / 위험 요인 / 확인 조건`을 카드 박스나 별도 소제목으로 쪼개지 않는다. 문단 흐름 안에서 자연스럽게 연결한다.

## 카피 제한

- 헤드라인: 최대 2줄
- 평가 라벨: 4~8자 권장
- 종합 평가: 최대 2줄
- 판단 기준: `기대할 것 / 확인할 것 / 판단할 것`을 빈 줄 없이 연속 3행으로 배치
- 핵심 본문: 3문단, 전체 6~9줄
- 최종 평가: 한 문장
- 투자 점수, 목표가, 매수·매도 신호 생성 금지
- 재무 효과가 확인되지 않았으면 확정적인 호재·악재 표현 금지

## 저널리스트 문체

- 평가를 먼저 제시하고 `왜 그렇게 보는가 → 무엇이 부족한가 → 무엇이 확인되면 평가가 달라지는가`로 전개한다.
- `긍정적이다`, `확인이 필요하다` 같은 보고서 문장을 반복하지 않는다.
- `그러나 관심이 곧 수익은 아니다`, `평가를 끌어올릴 조건은 단순하다` 같은 전환 문장으로 문단을 연결한다.
- 짧은 단정형, 질문형, 인용부호를 제한적으로 섞되 사실 판단을 과장하지 않는다.
- 마지막에는 기사 요약이 아니라 평가를 바꿀 핵심 데이터가 무엇인지 남긴다.

## 디자인 규칙

- 1·2페이지와 동일한 배경 이미지, 남색 오버레이, 우측 적색 색감을 사용한다.
- 제목은 `Noto Sans KR Black`, 본문은 `Noto Sans KR`을 사용한다.
- 카드 박스와 긴 세로 포인트 막대는 사용하지 않는다.
- 짧은 파란 가로선과 평가 라벨만 포인트로 사용한다.
- 평가 라벨 색상은 다음처럼 자동 결정한다.
  - 호재 계열: 청록 또는 파랑
  - 중립: 회색
  - 악재 계열: 빨강
- 제목 영역은 글자 하단 잘림 방지를 위해 최소 20px의 추가 높이를 확보한다.
- 판단 기준 3행 사이에는 빈 줄을 넣지 않는다. 확보한 세로 공간은 아래 분석 본문에 우선 배정한다.
- 분석 본문의 마지막 문장과 하단 핵심 결론 사이에는 최소 60px의 안전 여백을 둔다.

## 자동 검수

- 평가 강도와 기사 근거가 일치하는가
- 종합 평가가 뉴스 요약을 단순 반복하지 않는가
- 긍정 요인과 위험 요인이 모두 포함됐는가
- 판단을 바꿀 확인 조건이 구체적인가
- 점수·목표가·매수·매도 권유가 없는가
- 카드 박스와 긴 세로 막대가 없는가
- 제목과 본문이 잘리지 않는가
- 푸터가 이전 페이지와 동일한 위치인가
