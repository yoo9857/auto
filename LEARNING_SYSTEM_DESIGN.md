# 성장형 캐러셀 학습 설계서

이 시스템은 이전 게시물의 문구를 그대로 재사용하지 않는다. 대신 글 구성, 이미지 연출, 색상 전략의 성과를 기록하고 다음 기사에 적합한 전략을 선택한다.

## 단일 설계 기준점

모든 학습 규칙은 `automation/learning.design.json`에 있다. 전략 후보, 탐색률, 품질 가중치, 피드백 계약, 안전 제한을 수정하면 엔진 코드를 바꾸지 않고 다음 생성부터 적용된다.

## 운영 흐름

기사 추출 → 설계서 기반 전략 선택 → 글·이미지 생성 요청에 전략 전달 → 렌더·QA → 사람 검수 → 게시 성과와 편집 평가 기록 → 다음 기사 전략 선택

## 학습 원칙

- 사실 오류·권리 위험·오인 유발 사례는 성과가 좋아도 학습 데이터에서 제외한다.
- 품질 점수는 정확성 30%, 가독성 20%, 이미지 적합성 15%, 편집 승인 15%, 완독 10%, 참여 10%로 설계서에서 관리한다.
- 표본이 부족한 전략은 탐색 대상으로 남기고, 충분한 표본 뒤에만 성과 순위에 강하게 반영한다.
- 모든 선택 결과와 피드백 점수는 JSONL로 남아 재현·감사가 가능하다.

## 피드백 JSON 예시

```json
{
  "decision": "approved",
  "quality": {"accuracy": 5, "readability": 4, "image_fit": 5},
  "metrics": {"completion_rate": 72, "engagement_rate": 0.11},
  "notes": "제품 중심 이미지와 조건부 결론의 반응이 좋음"
}
```

## API 없이 운영하기

인스타 앱 또는 Meta Business Suite에서 게시물 인사이트를 열고 도달·좋아요·댓글·저장·공유·조회 수를 `automation/feedback.template.json`에 입력한다. 게시물 게시 후 48시간과 7일에 한 번씩 기록하면 API 없이도 전략 학습을 이어갈 수 있다. 공개 웹 스크래핑은 저장·공유·도달 정보를 제공하지 못하고 계정 정책 위험이 있으므로 사용하지 않는다.
