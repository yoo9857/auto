# 2026-07-15 N200 작업 기록

## 오늘 완료한 것

- 기사 링크 하나로 5장 캐러셀을 만드는 실행기 `build_n200_full.ps1`를 구성했다.
- 결과 순서: 1 훅 이미지 → 2 영상 → 3 분석 → 4 판단 → 5 마무리.
- 최종 게시 파일은 `DB\날짜-기사ID\01~05`에 정리한다.
- 외부 영상은 화면 오른쪽 하단에 `(c) 채널명`을 표기하고, 출처·사용 구간·권리 상태를 작업 JSON에 남긴다.

## 영상 선택 규칙

1. 기사 사건을 직접 보여 주는 영상이 최우선이다.
2. 직접 증거가 없을 때만 인물 컨텍스트 영상을 사용한다.
3. 인물 이름이 자막에 등장하는 것만으로는 통과하지 않는다.
   - 진행자 소개·질문 장면은 탈락시킨다.
   - 본인 자기소개 또는 본인 발언 신호를 우선한다.
4. 전체 메타데이터로 게시일을 재확인한다.
   - 최근 12개월을 기본 창으로 사용한다.
   - 검색 스니펫에 날짜가 없으면 최신 영상과 동등하게 평가하지 않는다.
5. 화면 인물 확인이 불완전하면 `needs_clip_review` 상태로 멈춘다.

## 디자인·카피 규칙

- 1080×1350, 다크 네이비 N200 팔레트, 일관된 로고·푸터·페이지 번호를 사용한다.
- 1페이지 대표 이미지는 참고용이다. 이미지 안에 글자·로고·기존 카드 UI가 있으면 그대로 쓰지 않는다.
- 1페이지는 텍스트 없는 N200 오리지널 배경과 두 줄 이내 제목, 두 줄 이내 부제를 사용한다.
- 2페이지는 H200과 같은 비디오 템플릿을 사용한다.
  - 위: 원본 비율을 보존한 영상
  - 가운데: 출처·저작권 스트립
  - 아래: 기사로 이어지는 `HOT ISSUE`/시장 신호 카피
- 3페이지는 원문 반복이 아닌 `변화 → 가치 → 순서` 분석 구조를 사용한다.
