# OneDayTrading 정독률 중심 콘텐츠 알고리즘

## 목표

낚시성 표현 없이 첫 1초의 관심, 페이지 전환, 마지막 장까지의 이해를 높인다. 한 페이지는 하나의 질문에만 답하며 같은 결론을 반복하지 않는다.

## 페이지별 역할

```text
1페이지: 무슨 일이 발생했나?
2페이지: 왜 중요한가?
3페이지: 그래서 어떻게 평가해야 하나?
4페이지: 어디에서 더 볼 수 있나?
```

## 생성 순서

### 1. 핵심 긴장 추출

기사에서 다음 대립쌍 하나를 찾는다.

- 기대 vs 현실
- 화제성 vs 실적
- 기회 vs 위험
- 발표 내용 vs 아직 공개되지 않은 내용
- 단기 반응 vs 장기 영향

한 기사에 대립쌍을 여러 개 사용하지 않는다.

### 2. 독자 질문 생성

독자가 실제로 궁금해할 질문을 18~28자로 만든다.

```text
유명 배우 한 명이 게임 매출까지 바꿀 수 있을까?
대규모 수주, 실제 이익은 언제 반영될까?
주가 급등은 실적인가 기대감인가?
```

질문은 2페이지 헤드라인 또는 리드문에 사용하고 같은 질문을 3페이지에서 반복하지 않는다.

### 3. 빠른 보상 제공

질문 직후 첫 문장은 반드시 잠정 답을 준다.

```text
가능성은 있다.
아직은 기대가 앞선다.
숫자만 보면 긍정적이다.
단기 호재지만 조건이 붙는다.
```

답을 숨겨 체류 시간을 늘리는 방식은 사용하지 않는다.

### 4. 근거 압축

근거는 최대 두 개만 남긴다.

- 확인된 사실 1개
- 아직 확인되지 않은 변수 1개

배경 설명, 같은 의미의 재진술, 상식적인 문장은 제거한다.

### 5. 판단 기준 제시

3페이지에는 독자가 이후 뉴스를 볼 때 사용할 기준 세 개만 남긴다.

```text
기대할 것
확인할 것
판단할 것
```

각 항목은 한 줄, 최대 24자로 작성한다. 별도 카드 박스 없이 기사 지면의 행 구조로 배치한다.

세 항목 사이에는 빈 줄을 넣지 않는다. 모바일에서 하나의 판단 묶음으로 인식되게 하고, 절약한 공간은 근거 설명에 사용한다.

### 6. 기억 문장 생성

페이지 하단에는 한 문장만 남긴다.

- 앞 문장을 단순 요약하지 않는다.
- 기사에서 가장 중요한 숫자·조건·변수를 강조한다.
- 최대 32자 권장

예시:

```text
시청률보다 중요한 숫자는 파크라이의 판매량이다.
수주 금액보다 먼저 볼 것은 영업이익률이다.
발표보다 중요한 건 실제 납품 일정이다.
```

## 분량 제한

```text
1페이지: 제목 2줄 + 서브타이틀 2줄
2페이지: 질문/헤드라인 2줄 + 본문 8~11줄 + 결론 1줄
3페이지: 평가 1줄 + 판단 기준 3줄 + 근거 설명 6~9줄 + 결론 1줄
4페이지: CTA 중심, 본문 최소화
```

## 문체 규칙

- 한 문장의 평균 길이는 18~32자로 유지한다.
- 긴 문장 다음에는 짧은 문장을 배치한다.
- 문단 첫 문장은 앞 문단의 결론을 받아야 한다.
- `하지만`, `그래서`, `결국`은 논리 전환이 있을 때만 사용한다.
- 질문형은 전체 캐러셀에서 최대 두 번 사용한다.
- 모든 문장을 `~다`로 끝내지 않고 명사형·질문형·짧은 단정형을 섞는다.
- 어려운 금융 용어는 일상어로 바꾸거나 즉시 설명한다.
- 정보가 부족해 보이지 않도록 각 페이지에 `확인된 사실`, `수익 또는 영향 경로`, `아직 남은 변수`를 최소 한 번씩 포함한다.
- 단, 근거를 늘리기 위해 원문에 없는 수치나 사례를 만들어내지 않는다.

## 중복 제거

생성 후 페이지별 핵심 문장을 비교한다.

- 의미 유사도가 높은 문장이 두 페이지에 있으면 하나를 삭제한다.
- 2페이지는 원인과 의미, 3페이지는 평가와 확인 데이터만 남긴다.
- 제목에서 말한 사실을 본문 첫 문장에서 그대로 반복하지 않는다.

## 자동 품질 점검

각 항목을 0 또는 1로 평가하며 8점 미만이면 다시 작성한다.

```text
1. 첫 2줄 안에 독자 질문 또는 긴장이 있는가
2. 질문 직후 잠정 답을 제공하는가
3. 한 페이지가 하나의 질문에만 답하는가
4. 확인된 사실과 미확인 변수가 구분되는가
5. 2·3페이지의 결론이 중복되지 않는가
6. 모바일에서 3초 안에 구조를 파악할 수 있는가
7. 기억 문장이 구체적인 판단 기준을 남기는가
8. 과장·공포·매수·매도 권유가 없는가
9. 기사 원문과 모든 사실이 일치하는가
10. 다음 페이지를 읽어야 할 이유가 자연스럽게 남는가
11. 마지막 본문 문장이 잘리거나 푸터와 충돌하지 않는가
12. 판단 기준의 여백이 본문보다 과도하게 넓지 않은가
```
