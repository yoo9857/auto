# OneDayTrading 자동 생성 시스템 설계서

## 목적과 범위

기사 URL 한 개로 4페이지 뉴스 캐러셀 초안, 근거 기록, 품질 검사, 사람 승인, 게시 성과 기반 전략 학습을 연결한다. 이 시스템은 투자 조언을 자동 게시하지 않으며, 항상 편집 승인 뒤에만 게시 가능한 상태가 된다.

## 상태 전이

```text
needs_image → draft → ready → approved
                      └→ rejected
```

- `needs_image`: 카피 또는 이미지가 미완성인 초안
- `draft`: 모든 입력은 있으나 검수 전
- `ready`: 자동 QA 통과, 편집 승인을 기다리는 상태
- `approved`: 사람이 승인한 게시 가능 상태
- `rejected`: 사실·권리·품질 문제로 학습 및 게시에서 제외

## 계약

1. `source.json`은 원문 제목·설명·본문 해시·수집 시간을 보존한다.
2. 카피 제공자는 페이지 1~4의 필수 필드를 모두 반환해야 한다.
3. 이미지 제공자는 이미지 경로·출처·라이선스 상태를 반환하거나 `review_required`로 남긴다.
4. `job.json`은 원문, 근거, 시각 자산, 선택 전략, 카피를 하나의 작업 단위로 묶는다.
5. `qa-report.json`이 통과해도 `approved`가 되지는 않는다.
6. 사람 검수 뒤의 피드백만 학습 저장소에 기록한다.

## Instagram 계정 매칭

브랜드 계정은 `@onedaytrading.io`와 `https://www.instagram.com/onedaytrading.io/`로 고정한다. 각 작업에는 `ODT-<article-id>` 추적 코드가 부여되며, 게시 뒤 Media ID를 연결하면 해당 게시물의 성과만 같은 기사에 반영한다. 계정 로그인 토큰은 파일에 저장하지 않고 환경 변수로만 사용한다.

## 품질 게이트

| 게이트 | 실패 시 처리 |
| --- | --- |
| 원문 URL·스냅샷 일치 | 렌더링 중단 |
| 카피 계약·문단 수 | 렌더링 중단 |
| 이미지 파일·비율·라이선스 | `draft` 유지, 승인 차단 |
| 사실·권리·오인 위험 | `rejected`, 학습 제외 |
| 레이아웃·텍스트·계정 QA | 렌더링 중단 또는 재작성 |
| 사람 승인 | 게시 및 성과 학습 허용 |

## 학습 경계

학습 엔진은 `automation/learning.design.json`만 해석한다. 전략과 평가 가중치는 설계서에서 변경하며, 모델 가중치를 임의로 수정하거나 원문 문장을 재사용하지 않는다.
