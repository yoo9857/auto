# OneDayTrading 두 번째 페이지 워크플로우

## 목적

첫 번째 속보 표지 다음에 배치되는 실제 경제 뉴스 분석면이다. 카드형 정보 박스나 `전망 / 요청 / 확장` 소제목을 사용하지 않고, 하나의 자연스러운 기사 본문으로 전달한다.

이 페이지는 오직 `왜 중요한가?`에 답한다. 종합 평가는 3페이지로 넘기며 같은 판단을 반복하지 않는다. 세부 생성 규칙은 `ENGAGEMENT_ALGORITHM.md`를 따른다.

## 입력 데이터

```json
{
  "article_title": "",
  "page2_headline": "",
  "lead": "",
  "expected_effect": "",
  "verification_points": "",
  "expansion_potential": "",
  "closing_sentence": "",
  "category": "",
  "market_label": "해외주식",
  "background_image": "",
  "logo": "NEWLOGO.png"
}
```

## 콘텐츠 생성 순서

1. 기사 원문에서 확인된 사실만 추출한다.
2. 2페이지용 분석 헤드라인을 새로 작성한다.
3. 기사 중요성을 설명하는 리드문을 2줄로 작성한다.
4. 다음 내용을 하나의 연속된 분석 기사로 합친다.
   - 기대 효과와 전망
   - 추가 공개 또는 확인이 필요한 변수
   - 산업·매출·다른 IP로의 확장 가능성
5. 마지막에 전체 분석을 압축한 핵심 결론 한 줄을 작성한다.

본문에 `전망`, `요청`, `확장`이라는 소제목은 표시하지 않는다. 문단만 구분하며 의미가 자연스럽게 이어지게 한다.

## 카피 제한

- 헤드라인: 최대 2줄
- 리드문: 최대 2줄
- 분석 본문: 3문단, 전체 8~11줄
- 각 문단: 2~3문장으로 구성하며 `사실/효과`, `수익 경로/미확인 변수`, `확장/실패 가능성`을 각각 담당
- 핵심 결론: 1문장
- 확인되지 않은 수치·일정·인과관계 금지
- 매수·매도 권유 표현 금지

## 저널리스트 문체

- 문단을 독립 정보로 나열하지 않고 `사실 → 반전/한계 → 확인 변수 → 의미`로 연결한다.
- 모든 문장을 기계적으로 `~다`로 끝내지 않는다.
- `문제는 그다음`, `그래서 먼저 볼 것은`, `이야기는 달라진다`, `결국` 같은 연결 문장을 상황에 맞게 사용한다.
- 질문형, 명사형, 짧은 단정형을 문단당 최대 한 번 섞어 리듬을 만든다.
- 수사적 질문은 답이 본문에 있을 때만 사용하며 선정적으로 쓰지 않는다.
- 마지막 문장은 요약을 반복하지 않고 투자자가 기억할 판단 기준을 남긴다.

## 디자인 규칙

- 첫 페이지와 동일한 배경 이미지를 사용한다.
- 배경 위에 짙은 남색 오버레이를 적용해 본문 가독성을 확보한다.
- 오른쪽에는 첫 페이지와 이어지는 절제된 적색 분위기를 유지한다.
- 카드 박스, 둥근 패널, 긴 파란 세로 막대는 사용하지 않는다.
- 헤드라인 아래의 짧은 파란 가로선만 브랜드 포인트로 사용한다.
- 제목: `Noto Sans KR Black`
- 본문: `Noto Sans KR`
- 제목 글자 하단이 잘리지 않도록 제목 영역에 최소 20px 이상의 여유를 둔다.
- 인물 얼굴은 배경으로만 약하게 남기고 본문보다 시각적 우선순위가 높아지지 않게 한다.

## 푸터 고정 좌표

첫 페이지와 완전히 동일한 기준을 사용한다.

```text
구분선 Y: 1207
NEWLOGO X: 72, Y: 1231, 크기: 58×58
사이트/시장 X: 151, Y: 1242
페이지 번호: 오른쪽, Y: 1242
투자 고지: 오른쪽, Y: 1276
```

푸터 문구:

```text
onedaytrading.net | {market_label}
02 / 04
투자 참고용 · 매매 권유 아님
```

## 자동 검수

- 헤드라인 하단 획이 잘리지 않았는가
- 리드문과 본문이 얼굴 때문에 읽기 어려워지지 않았는가
- 본문이 정확히 3문단으로 보이는가
- `전망 / 요청 / 확장` 소제목이 노출되지 않았는가
- 카드형 박스와 긴 파란 세로 막대가 없는가
- 핵심 결론이 한 줄 안에 들어가는가
- 푸터 좌표가 1페이지와 동일한가
- `NEWLOGO`, 시장 라벨, 페이지 번호가 정확한가
- 모든 문장이 기사 원문과 모순되지 않는가
